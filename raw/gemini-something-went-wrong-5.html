제미나이 “문제가 발생했습니다 (5)” 오류는
특별히 “원인이 이거다!” 하고 하나로 짚기 어려운 오류입니다.

연결 상태, 계정 세션, 모델 처리, 대화방 상태 중 하나가 정상적으로 맞물리지 않을 때 나타나는 포괄적인 오류에 가깝습니다.

한줄 결론
제미나이 “문제가 발생했습니다 (5)” 오류가 뜨면, 모델 변경 → 계정 확인 → 새 채팅 → 요청 줄이기 순서로 해결하는 게 제일 빠릅니다.

1. 먼저 Flash 모델로 바꿔보세요

제미나이 Pro에서 “문제가 발생했습니다 (5)” 오류가 뜬다면, 먼저 모델을 Flash로 바꿔서 같은 질문을 다시 보내보세요.

Pro를 계속 붙잡고 새로고침하는 것보다, Flash로 우회하는 편이 빠를 때가 있습니다. Flash에서 답변이 정상적으로 나온다면 급한 작업은 일단 이어갈 수 있고, Pro 모델은 잠시 뒤 다시 시도하면 됩니다.

2. 유료 계정으로 접속됐는지 확인하세요

구글 계정을 여러 개 로그인해둔 상태라면, 제미나이가 내가 원한 계정이 아니라 다른 계정으로 열릴 수 있습니다. 이럴 때는 authuser URL 파라미터로 접속 계정을 직접 지정해볼 수 있습니다.

authuser는 구글 계정을 여러 개 로그인했을 때, 어떤 계정으로 접속할지 지정하는 URL 파라미터입니다.

예를 들어 아래 주소는 제미나이를 0번 계정으로 열어달라는 뜻입니다.

https://gemini.google.com/?authuser=0

다만 0번이 무조건 유료 계정이라는 뜻은 아닙니다. 현재 브라우저에 로그인된 구글 계정 순서에 따라 0번, 1번, 2번이 달라집니다.

그래서 아래 주소를 하나씩 열어보고, 오른쪽 위 프로필 아이콘에서 어떤 이메일로 접속됐는지 확인해보세요.

https://gemini.google.com/?authuser=0
https://gemini.google.com/?authuser=1
https://gemini.google.com/?authuser=2

유료 플랜이 붙은 계정이 authuser=1에서 열린다면, 앞으로는 아래 주소로 접속하면 됩니다.

https://gemini.google.com/?authuser=1

그러면 제미나이가 구독한 계정으로 열릴 거예요.

3. 새 채팅에서 다시 보내보세요

같은 대화방에서 계속 오류가 반복된다면, 새 채팅을 열고 같은 질문을 다시 보내보세요.

기존 대화방이 너무 길거나, 이전에 이미지·파일·긴 코드 같은 요청이 섞여 있었다면 특정 대화방에서만 오류가 반복될 수 있습니다.

새 채팅에서는 정상적으로 답변이 나온다면, 기존 대화방보다 새 대화방에서 이어가는 편이 빠릅니다.

4. 요청을 짧게 나눠서 보내세요

긴 글, 긴 코드, 파일, 이미지까지 한 번에 넣었을 때 오류가 났다면 요청 범위를 줄여보세요.

예를 들면 이런 식입니다.

전체 파일 요약해줘
→ 첫 번째 부분만 먼저 요약해줘

긴 코드 전체 고쳐줘
→ 오류 나는 부분만 먼저 봐줘

요청이 길수록 처리 과정에서 오류가 날 가능성이 커집니다. 처음부터 전체를 맡기기보다, 작은 단위로 나눠서 보내는 편이 안정적입니다.

5. 이미지나 파일을 빼고 텍스트만 보내보세요

이미지나 파일을 첨부했을 때만 “문제가 발생했습니다 (5)”가 뜬다면, 먼저 텍스트 질문만 보내보세요.

텍스트 질문은 정상적으로 처리되는데 파일이나 이미지 요청에서만 오류가 난다면, 첨부파일 처리나 이미지 분석 과정에서 문제가 생긴 것일 수 있습니다.

이럴 때는 파일을 다시 올리기보다, 핵심 내용만 텍스트로 복사해서 보내는 방식이 더 빠를 수 있습니다.

6. Google Workspace 연결 기능을 잠깐 꺼보세요

Gmail, Drive, Docs 같은 자료를 읽어달라고 했을 때만 오류가 난다면, Gemini의 Google Workspace 연결 기능을 잠시 꺼보는 것도 방법입니다.

외부 자료를 불러오는 요청에서만 오류가 반복된다면, 제미나이 자체보다 연결 기능 쪽에서 문제가 생긴 것일 수 있습니다.

Workspace 연결을 잠시 끄고 일반 질문을 보내본 뒤, 다시 켜서 확인해보세요.

7. Gemini 앱 활동을 껐다가 다시 켜보세요

계정 쪽 설정이 문제처럼 느껴진다면, Gemini 앱 활동을 껐다가 다시 켜볼 수 있습니다.

myactivity.google.com → Gemini 앱 활동 → 끄기 → 1분 정도 기다리기 → 다시 켜기

이 방법은 브라우저 캐시를 지우는 것과는 다르게, Google 계정 쪽의 Gemini 활동 설정을 다시 확인하는 방식입니다.

다만 모든 오류를 해결하는 만능 방법은 아닙니다. 위 방법들을 해도 계속 오류가 날 때 추가로 시도해볼 수 있는 점검 정도로 보면 됩니다.

8. 그래도 안 되면 시크릿 모드, 캐시 삭제, 다른 브라우저를 확인하세요

위 방법을 모두 해도 계속 오류가 난다면 그때 브라우저 쪽을 확인해보세요.

시크릿 모드에서 Gemini가 정상 작동한다면 기존 브라우저의 쿠키, 캐시, 확장 프로그램 영향일 수 있습니다.

Chrome 기준 캐시와 쿠키 삭제 경로는 아래와 같습니다.

설정 → 개인정보 보호 및 보안 → 인터넷 사용 기록 삭제 → 쿠키 및 기타 사이트 데이터 / 캐시된 이미지 및 파일

또는 Edge, Safari, Firefox처럼 다른 브라우저에서 접속해보는 것도 방법입니다.

9. 계속 안 되면 1~2시간 뒤 다시 시도하세요

여기까지 했는데도 계속 “문제가 발생했습니다 (5)”가 뜬다면, 내 설정 문제가 아니라 제미나이 쪽 서버나 특정 모델 문제일 수 있습니다.

유저는 죄가 없습니다. 제미나이 쪽 문제인지 확인하고 싶다면 아래 상태 페이지도 같이 확인해보세요.

https://aistudio.google.com/status
https://www.google.com/appsstatus/dashboard/

상태 페이지에도 문제가 보인다면, 더 붙잡고 있을 필요 없습니다. 1~2시간 뒤에 다시 확인하는 게 제일 낫습니다.
