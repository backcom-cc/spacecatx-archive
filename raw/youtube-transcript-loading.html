유튜브 스크립트 무한로딩, 겪어보신 적 있으시죠?

강좌 영상 빠르게 훑어보고 싶을 때, 아니면 외국어 영상이라 전체 스크립트를 눈으로 쭉 보고 싶을 때 다들 한 번쯤 겪어보셨죠.

저도 어제 밤에 영어 공부한다고 유튜브에서 영어 강의 영상 하나 틀었거든요. 대충 흘려듣기만 하면 하나도 안 남을 것 같아서 자막 스크립트 좀 보려고 설명란 밑에 스크립트 표시를 눌렀는데, 이게 로딩만 뺑글뺑글 돌고 안 뜨더라고요.

새로고침도 해보고 다시 눌러보기도 하고, 몇 번을 반복했는지 모르겠어요.

원인부터 뒤지고 있기엔 너무 급했어서, 일단 대본부터 확인할 수 있는 방법 두 가지를 먼저 써봤어요. 순서대로 정리해드릴게요.

유튜브 스크립트가 계속 로딩만 된다면, 원인을 찾기 전에 먼저 아래 두 가지를 해보는 게 빠릅니다.

방법 1. 시크릿창 또는 로그아웃 상태로 다시 열기

같은 영상 링크를 복사해서 시크릿창에서 열어봤어요. 신기하게 로그인된 화면에서는 계속 로딩만 되던 스크립트가, 시크릿창에서는 바로 뜨더라고요.

PC 크롬 기준으로는 Ctrl + Shift + N을 누르면 시크릿창이 바로 열려요. 거기에 문제 생긴 영상 주소를 붙여넣고, 스크립트 표시를 다시 눌러보시면 됩니다.

모바일 앱에서 보고 있었다면 좀 번거로워요. 유튜브 앱에서 영상 공유 버튼을 눌러서 링크를 복사한 다음, 크롬이나 사파리 새 탭에서 열어서 확인해야 하거든요. 시크릿창까지 열기 귀찮으시면 유튜브에서 잠깐 로그아웃하고 같은 영상 보는 것도 방법이에요.

방법 2. 스크립트 추출 사이트에서 먼저 뽑기

당장 대본이 급하면 유튜브 안에서 붙잡고 씨름하는 것보다 이쪽이 더 빠를 때가 많아요. 저는 유튜브에서는 안 뜨던 영상도 youtubetotranscript.com에 링크 넣었더니 바로 대본이 나오더라고요.

방법은 간단해요.

  • 유튜브 영상에서 공유 버튼을 누릅니다.
  • 링크를 복사합니다.
  • youtubetotranscript.com에 들어갑니다.
  • 복사한 링크를 붙여넣고 transcript를 확인합니다.

유튜브 앱으로 보고 계셨다면 오히려 이 방법이 시크릿창 여는 것보다 덜 번거로울 수 있어요.

그래도 유튜브 안에서 해결하고 싶다면

위 두 방법은 어디까지나 급할 때 쓰는 우회로예요. 매번 시크릿창 열고 링크 복사하고 붙여넣는 것도 은근히 손이 가서, 가능하면 유튜브 안에서 바로 뜨는 게 제일 편하긴 하죠. 아래 방법들도 한 번씩 시도해보세요.

설명란 접었다가 다시 열어보기

해외 커뮤니티 보니까 스크립트 표시를 눌렀다가 설명란을 접고, 다시 펼쳐서 스크립트 버튼을 눌렀더니 대본이 나타났다는 사례가 있더라고요. 설명란이나 스크립트 영역이 다시 불러와지면서 멈춰있던 로딩이 한 번 새로고침되는 느낌인 것 같아요.

다만 PC에서는 스크립트 패널이 영상 옆에 따로 뜨는 경우가 많아서, 모든 상황에 통하는 방법이라기보다는 설명란 쪽 화면이 엉켰을 때 시도해볼 만한 정도로 봐주세요.

광고 차단기나 유튜브 관련 확장 프로그램 끄기

PC 브라우저로 보고 계시면 광고 차단기, 번역, 자막, 유튜브 UI 변경 확장 프로그램도 의심해볼 수 있어요.

유튜브는 댓글창, 추천 영상, 챕터, 스크립트 패널을 각각 따로 불러오는 구조라, 확장 프로그램이 화면 일부를 건드리면 스크립트 패널만 이상하게 로딩이 안 걸릴 수 있거든요. 댓글창도 같이 안 뜨거나 추천 영상 영역이 이상하다면 확장 프로그램 쪽을 먼저 의심해보세요.

youtube.com 사이트 데이터 삭제하기

시크릿창에서는 되는데 평소 쓰는 브라우저에서는 계속 안 된다면, 브라우저에 저장된 유튜브 사이트 데이터가 엉켰을 가능성이 있어요. 크롬 주소창에 아래 주소를 입력하면 사이트 데이터 관리 화면으로 바로 이동합니다.

chrome://settings/siteData

여기서 검색창에 youtube를 입력하고, youtube.com 관련 데이터만 삭제하면 됩니다. 전체 캐시랑 쿠키를 다 지우면 다른 사이트 로그인까지 풀릴 수 있으니, 처음부터 전부 지우기보다는 유튜브 데이터만 먼저 삭제해보는 쪽이 덜 번거로워요.

혹시 위 주소에서 화면이 바로 안 뜨면 아래 주소도 확인해보세요.

chrome://settings/content/all

여기서도 검색창에 youtube를 입력하면 관련 사이트 데이터를 찾을 수 있어요.

왜 이런 일이 생길까

정확한 원인 하나로 딱 정리하기는 어려워요. 같은 영상인데도 로그인 상태에서는 안 뜨고 시크릿창에서는 뜨는 경우가 있었고, 또 어떤 영상은 유튜브 안에서는 계속 로딩만 됐는데 추출 사이트에서는 대본이 바로 나오기도 했거든요. 아무래도 아래 원인들이 섞여있는 것 같아요.

로그인 세션이 엉킨 경우

로그인 상태에서는 계속 로딩되는데 시크릿창이나 로그아웃 상태에서는 정상적으로 뜬다면, 로그인 세션 쪽 문제일 가능성이 높아요.

영상 자체에 스크립트가 없어서가 아니라, 내 계정으로 보는 화면에서만 스크립트 패널이 제대로 안 불러와지는 상황에 가깝다고 보시면 됩니다. 그래서 시크릿창이나 로그아웃 상태로 열어보는 게 가장 빠른 구분법이었어요.

브라우저 확장 프로그램 영향

PC 브라우저로 볼 때는 광고 차단기, 번역, 자막, 유튜브 UI 변경 확장 프로그램도 영향을 줄 수 있어요.

댓글창, 추천 영상, 챕터, 스크립트 패널처럼 화면 안의 여러 영역을 유튜브가 각각 따로 불러오다 보니, 확장 프로그램이 그중 일부를 건드리면 스크립트 패널만 계속 로딩되는 식으로 문제가 생길 수 있습니다. 평소 브라우저에서는 안 되는데 시크릿창에서는 된다면 확장 프로그램도 한 번 의심해보세요.

자동 자막 처리 과정에서 늦게 뜨는 경우

업로더가 직접 자막을 등록하지 않은 영상은 유튜브가 음성을 인식해서 자동으로 자막을 만들어요. 이런 영상은 자막이나 스크립트가 바로 안정적으로 뜨지 않고, 인식하고 처리하는 과정에서 늦게 뜨거나 로딩이 길어질 수 있어요.

다만 이건 단정하긴 어려워요. 스크립트가 있는 영상도 안 뜰 때가 있고, 자동 자막 영상도 멀쩡하게 잘 뜰 때가 있거든요. 자동 자막 영상이라서 무조건 안 된다기보다는, 스크립트가 따로 등록되지 않은 영상이 조금 더 불안정하게 느껴질 수 있다 정도로 보시면 될 것 같아요.

유튜브 쪽 일시 오류나 UI 변경

여러 영상에서 갑자기 한꺼번에 스크립트가 안 뜬다면, 내 브라우저 문제가 아니라 유튜브 쪽 일시 오류일 수도 있어요.

유튜브가 설명란이나 스크립트 버튼 위치, 댓글창, 추천 영역 UI를 계속 바꾸다 보니 어느 날은 되던 기능이 갑자기 안 되는 것처럼 보일 때가 있거든요. 다른 브라우저나 다른 기기에서도 비슷하게 안 된다면, 내가 뭘 잘못한 게 아니라 잠깐 기다려야 하는 문제일 수도 있어요.
