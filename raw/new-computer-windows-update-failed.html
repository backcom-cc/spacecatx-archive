새 컴퓨터 윈도우 업데이트 실패..
새로 샀는데 왜 이래? 불량품을 받은 걸까?

이번에 새 노트북을 샀는데, 출고된 후 시간이 좀 지났는지 윈도우 업데이트가 1년 전 것부터 한꺼번에 쌓여 있더라고요.
이럴 때는 새 컴퓨터 자체 불량보다, 밀린 업데이트가 한꺼번에 설치되면서 꼬인 경우가 더 많습니다.

한 줄 결론: 새 컴퓨터에서 윈도우 업데이트가 실패하면, 먼저 날짜·시간 동기화 확인 → Windows 업데이트 문제 해결사 실행 → 재부팅 → 업데이트 확인 순서로 보는 게 가장 무난합니다. 특정 업데이트 하나만 계속 실패하면 Microsoft Update Catalog 에서 KB 번호를 직접 검색해 설치해볼 수도 있습니다.

새 컴퓨터 윈도우 업데이트 실패하는 이유?

새 컴퓨터라고 해서 항상 최신 상태로 오는 건 아닙니다.

출고 후 시간이 조금 지난 제품은 그동안 나온 윈도우 업데이트가 많이 쌓여 있을 수 있습니다. 그래서 처음 켜자마자 업데이트를 돌리면, 누적된 항목이 한꺼번에 설치되면서 오래 걸리거나 중간에 실패하는 경우가 있습니다.

또 의외로 많이 놓치는 게 날짜와 시간 설정입니다. 날짜나 시간이 현재와 맞지 않으면 업데이트 확인이나 설치 과정이 꼬일 수 있어서, 복잡한 작업 전에 이 부분부터 확인해보는 게 좋습니다.

해결 방법

1. 날짜와 시간부터 맞춰보기

가장 먼저 컴퓨터 오른쪽 아래 시간을 확인해보세요.

현재 날짜나 시간이 이상하게 맞지 않으면 업데이트 확인이나 설치 과정이 꼬일 수 있습니다.

경로는 아래와 같습니다.

설정 → 시간 및 언어 → 날짜 및 시간

여기서 아래 항목을 확인해보면 됩니다.

  • 시간 자동 설정
  • 시간대 자동 설정
  • 지금 동기화


2. Windows 업데이트 문제 해결사 실행

그다음은 가장 기본적인 방법입니다.

설정 → 시스템 → 문제 해결 → 기타 문제 해결사 → Windows 업데이트

여기서 문제 해결사를 실행하면, 윈도우가 스스로 업데이트 관련 문제를 점검하고 일부는 자동으로 수정해주는 경우가 있습니다.

3. 재부팅 후 업데이트 다시 확인

문제 해결사를 돌렸다면 컴퓨터를 한 번 재부팅한 뒤, 다시 업데이트를 확인해보는 게 좋습니다.

설정 → Windows 업데이트 → 업데이트 확인

처음 세팅하는 컴퓨터는 업데이트가 여러 개 연달아 잡히는 경우가 많아서, 한 번에 다 되지 않고 재부팅 후 다시 진행되는 경우도 흔합니다.

4. 그래도 안 되면 업데이트 캐시 정리

여기까지 했는데도 계속 실패한다면, 업데이트 캐시가 꼬였을 가능성도 있습니다.

이 경우에는 SoftwareDistribution 폴더 쪽 캐시를 정리해보는 방법이 자주 쓰입니다.

이 부분은 조금 길어질 수 있어서, 자세한 경로와 순서는 따로 정리한 글로 빼도 좋습니다.

5. 특정 KB만 계속 실패하면 수동 설치

이건 의외로 꽤 유용한 방법입니다.

업데이트 목록을 보면 가끔 특정 KB 번호 하나만 계속 실패하는 경우가 있습니다.

예를 들어 KB503… 같은 번호가 계속 실패한다면, 그 번호를 확인한 뒤 Microsoft Update Catalog 에서 직접 검색해서 수동 설치해볼 수 있습니다.

방법은 간단합니다.

1. Windows 업데이트 화면에서 실패하는 KB 번호 확인
2. Microsoft Update Catalog 접속
3. 검색창에 해당 KB 번호 입력
4. 내 PC 환경에 맞는 버전 다운로드 후 설치

다만 이 방법은 특정 업데이트 하나만 반복해서 실패할 때 더 잘 맞습니다.

여러 개가 한꺼번에 꼬인 상태라면, 문제 해결사나 캐시 정리를 먼저 해보는 쪽이 더 무난합니다.

새 컴퓨터 윈도우 업데이트 실패 – 마무리

새 컴퓨터 윈도우 업데이트 실패를 자꾸 겪으면… 제품 자체가 불량인가 싶어서 당황하게 됩니다.

그런데 실제로는 출고 후 밀린 업데이트가 한꺼번에 잡혀서, 또는 날짜·시간이나 업데이트 환경이 아직 정리되지 않아서 생기는 경우가 꽤 있습니다. Microsoft도 Windows 업데이트 문제에서 날짜·시간 확인, 문제 해결사 실행, 캐시 정리 같은 기본 점검 순서를 안내하고 있습니다.

특히 새 컴퓨터의 첫 업데이트는 생각보다 오래 걸릴 수 있습니다. 그래서 진행률이 바로 안 움직인다고 해서 곧바로 꺼버리기보다는, 잠깐 더 기다려보는 것도 필요합니다. 다만 같은 화면에서 너무 오래 멈춰 있거나 실패가 반복된다면, 그때는 재부팅 후 다시 확인하거나 문제 해결사로 넘어가는 쪽이 더 안전합니다.

그래서 처음부터 복잡한 복구를 하기보다는, 아래 순서로 차근차근 보는 게 좋습니다.

날짜·시간 확인 → 문제 해결사 실행 → 재부팅 → 업데이트 확인 → 필요하면 캐시 정리

그리고 업데이트 목록에서 특정 KB 번호 하나만 계속 실패한다면, 무조건 전체를 다시 건드리기보다 그 번호만 따로 보는 게 더 빠를 수 있습니다. 이럴 때는 Microsoft Update Catalog 에서 해당 KB 번호를 검색해 직접 설치해보는 방법도 꽤 유용합니다. 반대로 여러 업데이트가 한꺼번에 다 실패하는 상황이라면, 수동 설치보다 문제 해결사 실행이나 업데이트 캐시 정리를 먼저 해보는 쪽이 더 무난합니다. Microsoft Update Catalog는 Microsoft가 제공하는 공식 업데이트 카탈로그입니다.
