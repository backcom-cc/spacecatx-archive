구글 드라이브 액세스 필요 메세지.
분명 권한을 줬는데 왜 또 ‘액세스 필요’가 뜰까요?

구글 드라이브에서 파일을 공유했는데 상대방이 이렇게 말합니다.

“파일 열려고 했는데 액세스 권한이 필요하대요.”

분명 이메일도 정확하게 입력했고, 보기/편집 권한도 제대로 설정했습니다.

그런데도 상대방 화면에는 계속 ‘권한 요청’ 버튼만 보입니다.

“권한 요청 눌렀는데 또 같은 화면이 반복돼요.”

이런 상황이면 괜히 내가 설정을 잘못했나 싶어서 몇 번이나 다시 확인하게 되죠.

하지만 대부분은 설정 실수라기보다 권한 반영 지연이나 링크 캐시 문제인 경우가 많습니다.

바로 해결하는 방법

1. 파일 ‘사본 만들기’ 후 새 링크 공유

기존 파일을 우클릭 > 사본 만들기 > 새 파일을 다시 공유해보세요.
기존 링크가 아닌 “완전히 새로운 링크”를 만드는 것이 핵심입니다. 새 링크가 생성되면서 권한 정보가 새로 반영되는 경우가 많습니다.

2. 링크 공유 설정을 잠깐 전체 공개로 바꿨다가 되돌리기

공유 설정에서 “링크가 있는 모든 사용자”로 잠깐 변경한 뒤, 다시 특정 사용자 권한으로 되돌려보세요.
이 과정을 거치면 서버 권한 정보가 강제로 리프레시되는 효과가 있습니다.

3. 기존 권한을 완전히 제거 후 재부여

공유 대상에서 해당 사용자를 삭제한 뒤, 다시 이메일을 입력해서 권한을 재부여해보세요.
간단해 보이지만 실제로 가장 많이 해결되는 방법입니다.

4. 시크릿 모드에서 열어보기

브라우저 시크릿 모드로 링크를 열어보세요.
캐시 충돌인지, 로그인 계정 문제인지 빠르게 구분할 수 있습니다.

왜 이런 현상이 생길까?

구글 드라이브는 파일 권한 정보를 서버에서 동기화합니다.

간혹 다음과 같은 상황에서 오류처럼 보이는 문제가 발생할 수 있습니다.

  • 공유 드라이브(Shared Drive) 권한과 개별 파일 권한이 충돌
  • 링크 생성 직후 서버 인덱싱이 아직 반영되지 않음
  • 상대방이 다른 구글 계정으로 로그인한 상태
  • 브라우저 캐시가 이전 권한 정보를 유지하고 있음

“회사 계정으로 공유했는데, 상대가 개인 계정으로 로그인한 경우도 의외로 많습니다.”

이럴 경우 권한은 분명히 줬는데도 계속 접근이 막힌 것처럼 보일 수 있습니다.

그래도 안 된다면

권한 설정 직후라면 10~30분 정도 기다려보는 것도 방법입니다.

구글 서버 동기화가 지연되면 일시적으로 접근 오류가 발생할 수 있습니다.

대부분은 위 방법 중 하나로 해결됩니다.

정리

구글 드라이브에서 권한을 분명히 설정했는데도 “액세스 필요”가 반복된다면, 설정 오류보다는 권한 반영 지연, 링크 캐시, 계정 충돌 가능성이 큽니다.

파일을 새로 만들기 전에, 사본 생성 > 권한 토글 > 재부여 > 시크릿 모드 확인 순서로 점검해보세요.

괜히 파일 다시 업로드하지 마시고, 위 방법부터 차근차근 시도해보시면 됩니다.

Google Drive 고객센터의 Share files from Google Drive 안내처럼, 구글 드라이브 파일은 특정 사용자에게 보기·댓글·편집 권한을 줄 수 있고, 일반 액세스를 링크가 있는 사용자로 변경할 수도 있습니다. 권한을 줬는데도 “액세스 필요”가 뜬다면 먼저 공유 대상 계정과 링크 접근 설정을 다시 확인해보세요.
