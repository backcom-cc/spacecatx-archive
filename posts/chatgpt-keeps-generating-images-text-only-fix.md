## 챗지피티 자꾸 이미지 만듬. 새로 나온 이미지 모델을 자랑하고 싶은 걸까요?

한참 챗지피티랑 블로그 마스코트 외계고양이 스텔라 썸네일을 만들고 있었습니다. 요즘 이미지 모델이 업데이트되면서 퀄리티가 확 좋아졌거든요. 흐뭇하게 한참을 만들었습니다.

그런데 이미지 작업이 다 끝났는데도 이놈이 자꾸 이미지 캔버스를 꺼내는 겁니다.  
그냥 평상시 대화를 하는데도요.

![우리집 고양이는 상추 잘 먹더라. 그런데 고양이 육식만 하는거 아닌가? 
[이미지 생성 중..]](https://spacecatx.me/wp-content/uploads/2026/04/imga-1024x788.png)

고양이가 상추 먹어도 되는지 물어봤더니 이미지 생성 중… 한참을 돌립니다. 그러다가 갑자기 캔버스를 지우더니 아주 멀쩡한 답변을 내놓는 거예요.

*“고양이는 기본적으로 육식동물입니다. 상추를 아주 조금 씹는 정도는 괜찮을 수 있지만…”* 이런 식으로요.

지피티 본인도 모르겠는 겁니다. 붓을 들어야 할지 말아야 할지.

> **한 줄 결론:**   
> 챗지피티가 자꾸 이미지를 만들려고 하면 인스턴트 모드로 바꾸거나, “이미지 생성하지 말고 텍스트로만 답해줘”라고 먼저 못 박아보세요.

## 1. 지금 Thinking 모드라면 Instant 모드로 바꿔보기

챗지피티가 자꾸 이미지를 만들려고 할 때는, 먼저 지금 사용하는 모델이 **Thinking 모드인지** 확인해보세요.

![챗지피티 모드 설정](https://spacecatx.me/wp-content/uploads/2026/04/image-29.png)

저는 GPT를 거의 항상 Thinking 모드로 사용했기 때문에 이런 상황이 더 자주 생겼던 것 같습니다.

Thinking 모드는 질문에 바로 답하는 게 아니라, 대화 맥락 전체를 읽고 스스로 판단하는 과정을 거칩니다. 이미지 작업을 많이 한 채팅방이거나 이미지 작업을 자주한 사용자라면, 텍스트 질문에도 “이 사람은 이미지가 있으면 좋아할 것 같은데?” 하고 먼저 붓을 들 수 있는 거예요.

단순 답변, 문장 정리, 짧은 확인 질문이라면 **Instant** 모드로 바꿔보세요. 맥락을 덜 타기 때문에 이미지 없이 깔끔하게 텍스트로 답할 확률이 높아집니다.

---

## 2. 요청 앞에 “텍스트로만 답해줘”라고 쓰기

가장 먼저 해볼 방법은 질문 앞에 **“이미지 생성하지 말고 텍스트로만 답해줘”**라고 쓰는 겁니다.

```
이미지 생성하지 말고, 텍스트로만 답해줘.
```

조금 더 강하게 막고 싶다면 이렇게 써도 됩니다.

```
이미지 생성하지 말고, 도구 호출도 하지 말고, 한국어 텍스트로만 답해줘.
```

영어로는 이렇게 쓸 수 있습니다.

```
Do not generate images.
Do not call any tools.
Provide a text-only response in Korean.
```

다만 이 방법이 항상 먹히는 건 아니구요. 이미 이미지 작업방으로 굳어진 채팅에서는 챗지피티가 일단 캔버스부터 꺼내고 볼 때도 있습니다.

저도 “텍스트로만 답해줘”라고 했는데, 먼저 이미지 생성 화면을 띄운 뒤에야 멀쩡한 답변을 내놓는 경우가 있었습니다.

그래도 이 문구는 가장 간단한 1차 방어선입니다.  
여기서 안 먹히면 다음 단계로 넘어가면 됩니다.

---

## 3. 맞춤 지침에 이미지 생성 금지 문구 넣기

이 문제가 자주 반복된다면, 매번 요청 앞에 “이미지 만들지 마”라고 쓰는 것보다 **맞춤 지침**에 미리 넣어두는 편이 좋습니다.

맞춤 지침에는 아래 문구를 넣어두면 됩니다.

![챗지피티 맞춤형 지침](https://spacecatx.me/wp-content/uploads/2026/04/image-26-1024x910.png)

```
내가 명확하게 “이미지 만들어줘”, “그려줘”, “생성해줘”라고 요청하지 않는 한, 이미지 생성 도구를 사용하지 말고 텍스트로만 답해줘.
```

영어로 넣고 싶다면 이렇게 써도 됩니다.

```
Unless I explicitly ask to "generate an image", provide text-only responses.
```

다만 이 방법도 이미 이미지 작업을 오래 한 채팅방이라면 어려울 거예요. 이미 대화 흐름 자체가 이미지 생성에 초점이 맞춰져 있을지도 몰라요.

그래도 매번 “이미지 만들지 마”라고 쓰는 것보다는 훨씬 편합니다. 챗지피티에게 기본적으로 “내가 먼저 말하기 전까지 붓 들지 마”라고 알려두는 셈입니다.

---

## 4. 강력한 부정 프롬프트 쓰기

그래도 계속 이미지를 생성하려 덤벼든다면, 질문 앞에 조금 더 강한 문장을 붙여보세요.

먼저 한글로는 이렇게 쓸 수 있습니다.

```
이미지 생성하지 마.
도구 호출하지 마.
한국어 텍스트로만 답해.
```

이렇게 써도 의미는 전달됩니다.

그래도 챗지피티가 영어 기반으로 만들어진 서비스인 만큼, 짧은 영어 명령문이 더 명확하게 먹힐 때가 있습니다.

영어로는 이렇게 쓸 수 있습니다.

```
DO NOT generate images.
DO NOT call any tools.
Provide a text-only response in Korean.
```

대문자로 쓴다고 무조건 더 잘 듣는 건 아닙니다. 하지만 **“이번 요청은 이미지 생성이 절대 아니다”**라는 조건을 아주 선명하게 박아두는 효과는 있습니다.(아주 그냥 빨간 궁서체로 써주세요.)

---

## 5. 메모리에서 이미지 관련 기억 삭제하기

평소에 챗지피티로 이미지를 자주 만들었다면, **저장된 메모리 때문에 더 헷갈릴 수도 있습니다.**

예를 들어 챗지피티가 나를 이런 식으로 기억하고 있을 수 있어요.

> 이 사용자는 썸네일을 자주 만든다.  
> 이 사용자는 캐릭터 이미지를 자주 만든다.  
> 이 사용자는 시각 자료를 선호한다.

이런 기억이 남아 있으면, 평상시 대화에서도 챗지피티가 **“아, 이 사람은 이미지 만드는 거 좋아해! 만들어 주자!”** 하고 과하게 반응할 수 있습니다.

그래서 설정에서 메모리를 한 번 확인해보세요.

```
설정 > 개인화 > 메모리 관리
```

![챗지피티 개인 맞춤설정 메모리](https://spacecatx.me/wp-content/uploads/2026/04/image-27-1024x898.png)

여기에서 이미지 생성, 썸네일, 캐릭터 이미지, 시각 자료 선호 같은 기억이 있다면 삭제하면 됩니다.

다만 이미지를 자주 쓰는 사람이라면 관련 기억을 전부 지우는 게 나중에 아쉬울 수도 있습니다. 나중에 진짜 이미지 작업을 할 때 도움 되는 기억도 있으니까요.

## 눈치 없는 예술가 지피티, 그는 갑자기 왜 나대게 되었나?

![붓을 물고 급하게 달려오는 화가 베레모 스텔라](https://spacecatx.me/wp-content/uploads/2026/04/chatgpt-image-generation-rush-stella-819x1024.png)

*누구인가? 누가 이미지 소리를 내었어*?

2026년 4월 21일, ChatGPT Images 2.0이 업데이트됐습니다. 이때 같이 들어온 기능 중 하나가 **images with thinking**입니다. 이미지도 이제 생각하면서 만든다는 뜻인데요.

Thinking 모드에서는 대화의 맥락을 읽고, 스스로 이렇게 판단하는 거죠.

> “오, 이 채팅방은 이미지 작업하는 방이구나.”  
> “이 사용자는 그림이 있으면 좋아할 것 같아.”

AI 특유의 **도움이 되고자 하는 성향, Helpfulness** 때문입니다.

원래는 사용자를 도와주려고 하는 겁니다. 그런데 그 친절이 과해지는 거죠. 특히 이미지 작업 이야기를 많이 한 채팅방에서는 더 그럴 수 있습니다.

아마 지피티가 저를 거의 **“이미지 성애자”**로 보고 있었나 봅니다. 이미지란 단어는 물론, 썸네일, 디자인 이라는 소리만 내도 척수반사처럼 캔버스를 꺼내들죠.

다만 챗지피티는 사용자의 지적이나 부정적 피드백을 꽤 잘 반영하는 편입니다. 그래서 프롬프트나 맞춤 지침에 **“이미지 생성하지 말 것”** 같은 금지사항을 명확하게 적어두면, 이후에는 훨씬 조심해서 답할 가능성이 높습니다.
