##### 윈도우11 업데이트 실패 0x80248007 해결 방법

윈도우11 업데이트를 하다가 **“장치에서 중요한 보안 업데이트가 누락됐습니다 (0x80248007)”** 같은 문구가 뜨면서 업데이트가 실패하는 경우가 있습니다.

특히 새 컴퓨터를 샀거나, 오랜만에 업데이트를 몰아서 진행할 때도 이런 오류가 한 번씩 보입니다. 이럴 때는 괜히 이것저것 건드리기보다, **Windows 업데이트 문제 해결사 실행 → 업데이트 캐시 점검** 순서로 보는 게 제일 무난합니다.

---

> **한 줄 결론:** 가장 먼저 해볼 건 **Windows 업데이트 문제 해결사 실행**이고, 안 되면 **업데이트 캐시(SoftwareDistribution) 정리**까지 가는 게 제일 무난합니다. 그래도 해결되지 않으면 그때 **수동 초기화(catroot2 포함)** 쪽으로 넘어가면 됩니다.

## 0x80248007 오류는 뭔가요?

이 오류는 보통 **Windows Update가 필요한 파일이나 정보를 제대로 읽지 못할 때** 나타나는 편입니다.

쉽게 말하면

* 업데이트 캐시가 꼬였거나
* 관련 서비스가 잠깐 이상해졌거나
* 다운로드된 업데이트 파일이 정상적으로 처리되지 못한 경우

이런 쪽에서 자주 보입니다.

즉, 컴퓨터가 완전히 망가졌다기보다는 **업데이트 재료나 경로가 꼬여서 생기는 오류**에 더 가깝습니다.

---

## 윈도우11 업데이트 실패 0x80248007 해결 방법

### 1. Windows 업데이트 문제 해결사 실행

가장 먼저 자동 진단부터 해봅니다.

```
설정 → 시스템 → 문제 해결 → 기타 문제 해결사 → Windows 업데이트
```

이 단계에서 바로 해결되는 경우도 있어서, 괜히 복잡한 작업부터 하지 말고 먼저 돌려보는 걸 추천합니다.

### 2. PC 재부팅 후 다시 업데이트 확인

의외로 제일 간단한 방법으로 해결되는 경우도 있습니다.

* 컴퓨터를 한 번 재부팅하고
* `설정 → Windows 업데이트 → 업데이트 확인`을 다시 눌러보세요.

업데이트 파일이 반쯤 걸린 상태였던 경우에는 이것만으로 풀릴 때도 있습니다.

### 3. 업데이트 캐시 정리하기

이 오류는 **업데이트 임시 파일이 꼬였을 때**도 자주 보입니다.

조금 번거롭지만, 아래 순서대로 하면 됩니다.

#### 1) Windows Update 서비스 먼저 멈추기

키보드에서 `Win + R`을 누른 뒤 `services.msc`를 입력하고 엔터를 누릅니다.

서비스 목록에서 **Windows Update**를 찾아 마우스 오른쪽 클릭 후 **중지**를 누릅니다.

#### 2) 업데이트 캐시 폴더로 이동하기

파일 탐색기에서 아래 경로로 들어갑니다.

```
C:\Windows\SoftwareDistribution
```

여기 안에는 윈도우 업데이트가 내려받은 임시 파일들이 들어 있습니다.

#### 3) 안의 내용 삭제하기

**SoftwareDistribution 폴더 자체를 지우는 게 아니라**, 폴더 안에 들어 있는 파일과 폴더를 삭제하면 됩니다.

삭제가 안 되는 파일이 있으면, 서비스가 완전히 안 멈췄을 수 있으니 다시 `services.msc`로 가서 **Windows Update**가 정말 중지되었는지 확인해보세요.

#### 4) Windows Update 서비스 다시 시작하기

다시 서비스 창으로 돌아가서 **Windows Update**를 마우스 오른쪽 클릭한 뒤 **시작**을 누릅니다.

그다음 `설정 → Windows 업데이트 → 업데이트 확인`으로 다시 시도해보면 됩니다.

### 4. 관련 서비스 다시 확인하기

업데이트가 실패할 때는 관련 서비스가 비정상 상태인 경우도 있습니다.

확인해볼 서비스는 보통 이런 것들입니다.

* Windows Update
* Background Intelligent Transfer Service (BITS)
* Cryptographic Services

이 서비스들이 멈춰 있거나 꼬여 있으면 업데이트가 정상적으로 진행되지 않을 수 있습니다.

### 5. 그래도 안 되면: 조금 더 강한 초기화 방법

여기부터는 **문제 해결사 실행 → 재부팅 → 캐시 정리**까지 했는데도 안 될 때 시도하는 단계입니다.

관리자 권한 명령 프롬프트에서 아래 명령어를 한 줄씩 실행합니다.

```
net stop bits
net stop wuauserv
net stop cryptsvc
ren %Systemroot%\SoftwareDistribution\DataStore DataStore.bak
ren %Systemroot%\SoftwareDistribution\Download Download.bak
ren %Systemroot%\System32\catroot2 catroot2.bak
```

이건 일반 사용자에겐 조금 어렵게 느껴질 수 있으니, 본문에서는 **고급 방법** 또는 **그래도 안 될 때** 정도로 두는 게 좋습니다.

### 6. 윈도우11 업데이트 실패 0x80248007 해결 방법 – Microsoft 공식 안내도 같이 확인해보세요

위 방법으로도 해결되지 않으면 Microsoft 공식 Windows 업데이트 문제 해결 안내를 같이 보는 게 좋습니다.

[Microsoft 공식 Windows 업데이트 문제 해결 안내 보기](https://support.microsoft.com/en-us/windows/troubleshoot-problems-updating-windows-188c2b0f-10a7-d72f-65b8-32d177eb136c)

## 마무리

0x80248007 오류는 보통 하드웨어가 망가진 심각한 문제라기보다, **업데이트 캐시나 관련 서비스가 꼬였을 때** 나타나는 경우가 많습니다.

그래서 괜히 복잡한 복구부터 들어가기보다는, **문제 해결사 → 재부팅 → 업데이트 캐시 정리** 이 순서로 차근차근 해보는 게 좋습니다.

그리고 이 순서로도 해결되지 않을 때만 `catroot2`까지 포함한 조금 더 강한 수동 초기화를 시도하는 편이 안전합니다.

의외로 업데이트 관련 임시 파일만 정리해도 풀리는 경우가 있으니, 너무 겁먹지 말고 하나씩 해보세요.
