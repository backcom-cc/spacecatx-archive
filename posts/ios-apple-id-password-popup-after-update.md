# iOS 업데이트 후 “Apple ID 암호를 입력하십시오” 반복될 때

## iOS 업데이트 Apple ID 암호 입력 팝업이 계속 뜨나요?

미루고 미루다 겨우 iOS 업데이트를 했는데, 업데이트가 끝나자마자 이런 팝업이 계속 뜰 때가 있습니다.

> “Apple ID 암호를 입력하십시오.”

한 번이면 괜찮습니다. 그런데 닫으면 또 뜨고, 잠금 풀면 또 뜨고, 앱을 열면 또 뜨면 진짜 스트레스입니다.

이러면 괜히 계정이 털린 건가 싶고, 업데이트를 괜히 했나 싶고, 아이폰이 이상해진 것처럼 느껴집니다.

하지만 **iOS 업데이트 후 Apple ID 암호 입력** 팝업이 반복된다고 해서 바로 해킹이라고 단정할 필요는 없습니다. 업데이트 이후 iCloud, App Store, 미디어 및 구입 항목 계정의 재인증 과정이 꼬이면서 반복 팝업이 뜨는 경우가 있습니다.

다만 Apple ID는 계정 보안과 연결되어 있으니, 반복 팝업이 뜬다면 재로그인과 함께 기기 로그인 기록도 같이 확인하는 것이 좋습니다.

---

## 왜 iOS 업데이트 후에 이런 일이 생길까요?

iOS 업데이트를 하면 시스템이 새로 정리되면서 Apple ID와 연결된 여러 서비스를 다시 확인합니다.

* iCloud
* App Store
* 미디어 및 구입 항목
* iMessage
* FaceTime
* 사진, 메모, 연락처 같은 iCloud 동기화 항목

이 과정에서 인증이 한 번에 깔끔하게 끝나지 않으면, 아이폰이 계속 Apple ID 암호를 요구할 수 있습니다.

특히 아래 상황에서 반복 팝업이 생길 수 있습니다.

* iCloud 동기화가 아직 완료되지 않은 경우
* App Store 계정과 iCloud 계정이 다른 경우
* 최근 Apple ID 비밀번호를 변경한 경우
* 예전에 쓰던 Apple ID 정보가 일부 설정에 남아 있는 경우
* 업데이트 직후 Apple 서버 인증이 지연되는 경우
* 네트워크가 불안정해서 인증이 제대로 끝나지 않은 경우

그러니까 이 팝업은 “무조건 해킹”이라기보다, 업데이트 후 계정 인증과 동기화가 덜 끝난 상태일 수 있습니다.

---

## 먼저 Apple ID 계정 상태를 확인하세요

반복 팝업이 뜬다면 제일 먼저 현재 Apple ID가 정상 로그인 상태인지 확인해보세요.

```
설정 > 상단 Apple ID 이름
```

여기에서 이름, 이메일, iCloud 상태가 정상적으로 보이는지 확인합니다. 만약 “업데이트 필요”, “일부 계정 서비스 사용 불가”, “암호 입력 필요” 같은 안내가 보이면 해당 안내를 먼저 처리하는 것이 좋습니다.

이 단계에서 암호를 한 번 정확히 입력했는데도 계속 반복된다면, 아래 순서로 확인해보세요.

---

## 1. Apple ID 로그아웃 후 다시 로그인하기

반복 팝업이 계속 뜬다면 Apple ID에서 로그아웃한 뒤 다시 로그인하는 방법을 시도해볼 수 있습니다.

```
설정 > 상단 Apple ID 이름 > 로그아웃 > 다시 로그인
```

이 과정에서 iCloud 동기화가 다시 잡히면서 반복 팝업이 줄어드는 경우가 있습니다.

다만 로그아웃 전에는 꼭 확인해야 할 것이 있습니다.

* Apple ID 비밀번호를 정확히 알고 있는지
* 2단계 인증을 받을 수 있는 전화번호가 맞는지
* iCloud 사진, 메모, 연락처 동기화 상태가 괜찮은지
* 업무용 또는 가족 공유 계정이 아닌지

비밀번호가 확실하지 않은 상태에서 무작정 로그아웃하면, 다시 로그인하는 과정이 더 귀찮아질 수 있습니다. 먼저 비밀번호와 인증 수단을 확인한 뒤 진행하세요.

---

## 2. iCloud 동기화 항목을 껐다가 다시 켜보기

Apple ID 전체 로그아웃이 부담스럽다면, 먼저 iCloud 동기화 항목을 확인해보는 것도 방법입니다.

```
설정 > 상단 Apple ID 이름 > iCloud
```

여기에서 자주 사용하는 항목을 확인합니다.

* 사진
* iCloud Drive
* 연락처
* 캘린더
* 메모
* 미리 알림

특정 항목이 계속 동기화 중이거나 오류가 있는 것처럼 보인다면, 해당 항목을 잠시 껐다가 다시 켜보세요. 이 과정에서 iCloud 동기화가 다시 잡히는 경우가 있습니다.

다만 사진이나 iCloud Drive처럼 데이터가 큰 항목은 끄기 전에 안내 문구를 잘 읽어야 합니다. 기기에서 데이터를 삭제할지, 유지할지 묻는 경우가 있기 때문입니다.

---

## 3. 미디어 및 구입 항목 계정을 확인하세요

의외로 많이 놓치는 부분이 **미디어 및 구입 항목** 계정입니다.

iCloud에 로그인된 Apple ID와 App Store, 구입 항목에 연결된 Apple ID가 다르면 업데이트 후 암호 입력 팝업이 반복될 수 있습니다.

```
설정 > 상단 Apple ID 이름 > 미디어 및 구입 항목
```

여기에서 현재 어떤 계정으로 로그인되어 있는지 확인해보세요.

예전에 쓰던 Apple ID로 앱을 다운로드했거나, 가족 계정 또는 다른 국가 계정으로 구입한 앱이 있다면 해당 계정 암호를 요구할 수 있습니다.

이 경우 현재 쓰는 Apple ID 비밀번호를 아무리 입력해도 팝업이 반복될 수 있습니다. 어떤 계정의 암호를 요구하는지 이메일 주소를 꼭 확인하세요.

---

## 4. 와이파이 대신 모바일 데이터로 인증해보기

업데이트 직후 네트워크가 불안정하면 Apple ID 인증이 제대로 끝나지 않을 수 있습니다. 와이파이에서 계속 팝업이 반복된다면 모바일 데이터로 바꿔서 다시 시도해보세요.

* 와이파이 끄기
* 모바일 데이터 켜기
* Apple ID 암호 다시 입력
* 인증 완료 후 다시 와이파이 연결

반대로 모바일 데이터가 불안정하다면 안정적인 와이파이에서 다시 인증해보는 것도 좋습니다.

핵심은 네트워크를 바꿔서 Apple ID 인증 요청이 정상적으로 완료되는지 확인하는 것입니다.

---

## 5. iPhone을 재시동하기

계정 정보와 네트워크를 확인했는데도 계속 뜬다면, 아이폰을 한 번 재시동해보세요.

업데이트 직후에는 백그라운드에서 iCloud 동기화, 앱 업데이트, 계정 인증이 동시에 돌아가면서 팝업이 반복될 수 있습니다. 재시동 후 인증 상태가 정리되는 경우도 있습니다.

재시동 후에는 바로 여러 앱을 열기보다, 설정 앱에서 Apple ID 상태가 정상인지 먼저 확인하는 것이 좋습니다.

---

## 진짜 해킹일 수도 있나요?

대부분은 업데이트 후 재인증이나 동기화 문제일 수 있습니다. 하지만 Apple ID는 중요한 계정이기 때문에 아래 상황이라면 바로 보안 확인을 해야 합니다.

* 모르는 기기 로그인 알림이 온 경우
* 이상한 결제 알림이 함께 뜬 경우
* 내가 바꾸지 않았는데 비밀번호 변경 알림을 받은 경우
* Apple ID에 모르는 전화번호나 이메일이 추가된 경우
* 설정의 기기 목록에 모르는 기기가 보이는 경우

이런 경우에는 반복 팝업만의 문제가 아닐 수 있으니 Apple ID 비밀번호를 변경하고, 신뢰할 수 있는 기기 목록을 확인하는 것이 좋습니다.

```
설정 > 상단 Apple ID 이름 > 기기 목록 확인
```

모르는 기기가 있다면 계정에서 제거하고 Apple 지원에 문의하는 것이 안전합니다.

---

## 정리

iOS 업데이트 후 “Apple ID 암호를 입력하십시오” 팝업이 반복된다면, 대부분은 업데이트 이후 Apple ID 재인증이나 iCloud 동기화 과정이 꼬인 경우일 수 있습니다.

* Apple ID 상태 확인
* 로그아웃 후 다시 로그인
* iCloud 동기화 항목 확인
* 미디어 및 구입 항목 계정 확인
* 네트워크 변경 후 인증
* 아이폰 재시동
* 모르는 기기나 결제 알림이 있다면 보안 점검

**iOS 업데이트 후 Apple ID 암호 입력 팝업이 계속 뜬다면, 먼저 Apple ID와 미디어 및 구입 항목 계정이 같은지 확인하고, 네트워크를 바꿔 인증한 뒤에도 반복되면 로그아웃 후 재로그인을 시도해보세요.**

처음에는 계정이 털린 것 같아 불안하지만, 차분히 확인해보면 업데이트 후 인증이 덜 끝난 단순한 문제인 경우도 많습니다.

Apple ID 암호가 기억나지 않거나 로그인 자체가 되지 않는다면 [Apple 지원의 Apple ID 암호 재설정 안내](https://support.apple.com/ko-kr/102656)를 먼저 확인하세요. 반복 팝업이 단순 재인증 문제가 아니라 비밀번호 문제라면 암호 재설정이 필요할 수 있습니다.
