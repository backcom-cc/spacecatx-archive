### **KB5079473 삭제**는 윈도우11 업데이트가 99%에서 멈추거나 재부팅이 반복될 때 시도해볼 수 있는 임시 해결 방법입니다.

아니 진짜… 업데이트 한 번 하려다가 암걸리겠네요

> 99%에서 멈추고, 재부팅하면 또 업데이트,  
> 로그인도 안 되고 C드라이브까지 막혀버리면  
> 이건 단순 오류가 아니라 업데이트 충돌로 시스템이 꼬인 상태입니다.

---

## 결론

문제 업데이트(KB5079473)를 삭제하고 자동 업데이트를 일시적으로 차단하면  
대부분의 반복 오류는 해결됩니다.

---

## 적용 대상

다음과 같은 증상이 반복된다면 해당 문제일 가능성이 높습니다.

* 업데이트가 94~100%에서 멈추고 재부팅이 반복됨
* OneDrive, Teams, Microsoft 계정 로그인 실패 (0x800704cf)
* C드라이브 접근 거부 발생
* 업데이트 설치 실패 후 재시도 루프가 계속됨

---

## 1단계: KB5079473 강제 삭제

설정 메뉴에서 삭제가 되지 않는 경우가 많기 때문에  
명령어를 이용한 제거가 가장 확실합니다.

1. 작업표시줄 검색에서 **CMD** 입력 후 **관리자 권한으로 실행**
2. 아래 명령어를 입력한 뒤 실행

```
wusa /uninstall /kb:5079473
```

3. 제거가 완료되면 시스템을 재부팅합니다.

이 단계에서 정상 상태로 복구되는 경우가 많습니다.

---

## 2단계: 자동 업데이트 차단

문제 업데이트가 다시 설치되는 것을 막기 위해  
일시적으로 Windows Update 서비스를 중지합니다.

1. **Win + R** 키를 누른 뒤 `services.msc` 입력
2. **Windows Update** 항목 선택
3. **시작 유형**을 **사용 안 함**으로 변경
4. **중지** 클릭 후 **확인**

문제가 해결된 이후에는 다시 자동으로 변경하는 것을 권장합니다.

---

## 3단계: 추가 업데이트 수동 설치

해당 오류와 관련된 업데이트가 별도로 배포된 상태입니다.  
자동 업데이트에 반영되지 않는 경우 직접 설치가 더 빠를 수 있습니다.

**👉 [공식 MS 업데이트 카탈로그에서 KB5085516 다운로드](https://www.catalog.update.microsoft.com/Search.aspx?q=KB5085516)**

사용 중인 시스템 버전에 맞는 파일(x64 등)을 선택해 설치하면 됩니다.

---

## 추가 안내

위 방법으로도 해결되지 않는다면 윈도우 업데이트 구성 자체가 손상되었을 가능성이 있습니다.

이 경우에는 다음 단계로 진행해야 합니다.

* C드라이브 접근 거부 해결 방법
* 99% 멈춤 및 0x80070306 오류 해결 방법
* 파일 유지 상태에서 윈도우 복구 (인플레이스 설치)
